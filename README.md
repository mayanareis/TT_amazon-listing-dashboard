# Amazon Listing Intelligence Dashboard

A Streamlit dashboard for analysing Amazon product listings as a **competitive content intelligence tool**.

Instead of focusing on pricing analytics, this dashboard analyses *how* competitors write their titles and descriptions, what marketing claims they make, and how they present their products visually — giving you actionable insights for improving your own listings.

---

## What the dashboard does

| Tab | What you get |
|-----|-------------|
| **Overview** | Market snapshot — total products, average rating, review volume, brand distribution |
| **Title Intelligence** | Top keywords and phrases in competitor titles, title-length benchmarks, top-performing title examples |
| **Description Intelligence** | Theme extraction, marketing claim frequency, messaging category coverage (social proof, safety, comfort, convenience, design, research), and a Tommee Tippee vs competitor opportunity analysis |
| **Image Analysis** | Thumbnail gallery sorted by review count, brand image comparison |
| **Product Explorer** | Searchable, sortable product table with clickable Amazon links |
| **Tommee Tippee Deep Dive** | Full content analysis for Tommee Tippee listings only — title & description intelligence, competitor gap analysis, claim coverage score, and auto-generated messaging recommendations |

## Dataset structure

The dashboard reads `data/cleaned/amazon_products_clean.csv`, which is produced by `scripts/clean_data.py` from a raw JSON export.

Key columns used by the dashboard:

| Column | Type | Description |
|--------|------|-------------|
| `title` | string | Full product title |
| `clean_title` | string | Lowercased title (used for tokenisation) |
| `title_word_count` | int | Number of words in the title |
| `brand` | string | Brand name |
| `stars` | float | Average star rating (0–5) |
| `reviewsCount` | float | Number of customer reviews |
| `price_value` | float | Numeric price in USD |
| `description` | string | Full product description |
| `description_length` | int | Character length of description |
| `breadcrumbs` | string | Category path, e.g. `Baby Products > Feeding > ...` |
| `thumbnailImage` | string | Product thumbnail URL |
| `url` | string | Amazon product page URL |

## Setup and running locally

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/amazon_listing_dashboard.git
cd amazon_listing_dashboard
```

### 2. Create a virtual environment

```bash
python -m venv .venv
```

### 3. Activate the virtual environment

**Windows (PowerShell):**
```powershell
.\.venv\Scripts\Activate.ps1
```

If script execution is blocked, run once:
```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
```

**macOS / Linux:**
```bash
source .venv/bin/activate
```

### 4. Install dependencies

```bash
pip install -r requirements.txt
```

### 5. (Optional) Rebuild the cleaned dataset from raw data

If you have a raw JSON export from the Amazon scraper, place it in `data/raw/` and run:

```bash
python scripts/inspect_data.py   # inspect structure
python scripts/clean_data.py     # produce cleaned CSV
python scripts/validate_data.py  # verify output
```

### 6. Launch the dashboard

```bash
streamlit run dashboard/app.py
```

The dashboard opens at [http://localhost:8501](http://localhost:8501).

## Deploying to Streamlit Community Cloud

1. Push the repository to GitHub (raw data is excluded by `.gitignore`; the cleaned CSV is included).
2. Go to [share.streamlit.io](https://share.streamlit.io) and connect your GitHub account.
3. Set the **main file path** to `dashboard/app.py`.
4. Streamlit Cloud will install dependencies from `requirements.txt` automatically.
5. The `.streamlit/config.toml` configures the server for headless deployment.

## Notes

- Raw data is never overwritten by any script.
- The data pipeline handles nested JSON, missing fields, and malformed values.
- Price outliers (outside $3–$100) and non-baby-product categories are filtered out at load time to keep analysis relevant.
- All keyword extraction uses phrase-aware tokenisation so multi-word concepts like `anti-colic`, `BPA Free`, and `Tommee Tippee` are counted as single units rather than split into fragments.
